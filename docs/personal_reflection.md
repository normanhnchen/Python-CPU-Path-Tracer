 # My Personal Reflection

## What I Hoped To Do

My original goal was simple: create a ray tracer with realistic lighting. However as time went on,  
my goals gradually became more and more ambitious. Although the path tracer isn't perfect, I've  
learned a lot along the way.

## What I Learned (A Long List of Things)

Throughout my journey, I went through a lot of ideas and researched on a lot of things.

Even though I may not have implemented it in the final code, I've learned many new things:

### Examples:

### Math/Rendering

- Basic linear algebra and numpy vector code
- Screen space → world space projection
- Diffuse and specular reflections
- Fresnel Schlick reflectance
- Blinn-Phong and PBR materials
- Sphere intersection
- Möller–Trumbore intersection
- Slab method intersection

### Performance

- Numba `njit` and `jitclass` compiling
- CPU Thread Parallelization
- CPU Multiprocessing

### Tools / Workflow

- `.obj` files and `.mtl` files
- Markdown
- Basic Pygame creation
- Image saving using Pillow
- Bilateral filtering
- AI denoising
- Docstrings
- $\LaTeX$

## What I Found Challenging

The main bottlenecks of creating this project:

- Numba compiling errors
- Debugging intersection bugs
- Incorrect lighting calculations
- Numpy vector errors
- Multithreading / multiprocessing errors

## Conclusion

In the end, I created something I never thought I was capable of building. It was a difficult  
journey, and many things could still be improved greatly, but in the end I've created a Python CPU  
path tracer. I hope to continue this passion and build even more advanced projects in the future.
